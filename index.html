Accessibility Web Explainer

Easy best practices for a more accessible web

As of April of 2020, almost 4.57 billion people were active internet users worldwide. It is estimated that 1 billion people have some form of disability, many of which are also internet users. Based on these facts, it is our responsability as web developers to provide an accessible experience to as many individuals as we can.

Semantic tags

Hypertext Markup Language (HTML) is the standard markup language for documents designed to be displayed in a web browser. The HTML structure of a website is made up of tags. For example, the text you are reading right now is enclosed in a <p> tag, which stands for paragraph.

There are many other HTML tags, such as divs, buttons, and forms. HTML5, the latest version of HTML, introduced several additional "semantic elements", including article, aside, section, and more. These new elements are meant to clearly describe its meaning. This not only helps developers make sense of the HTML structure, but it also helps screen reader users navigate the page more effectively.

If you inspect the source code of this web page, you'll see that some of the semantic elements used are header, nav, main and footer.

Example

Non-semantic HTML structure

<div>
    <h1> </h1>
    <div </div> 
</div>
<div>
    <div> </div> 
    <div> </div> 
</div>
<div> </div>

Semantic HTML structure

<header>
    <h1> </h1>
    <nav </nav> 
</header>
<main>
    <section> </section> 
    <section> </section> 
</main>
<footer> </footer>

Alt text

Alt text (alternative text) is a word or phrase that can be inserted as an attribute in an HTML document to tell Web site viewers the nature or contents of an image. Including alt text for images is particularly important for screen reader users, who need a written description to avoid missing out on the graphic content.

In addition to being an absolutely necessity for accessibility, alt text acts as a fallback. If an image fails to load, alt text will guarantee that some text will be displayed instead.

:focus

Some HTML elements have a hover state to indicate to the users that the element is interactive. This is very commonly seen in elements such as anchors, which are used for links. Adding hover states to your site is generally considered a good practice, but there's a problem: it only works if you're using a mouse. Many users, including those with dexterity issues and other disabilities, exclusively navigate the web using their keyboard.

To solve the accessibility problem of hover states, we have the :focus pseudo-class. This CSS pseudo-class represents an element (such as a form input or an anchor) that has received focus. Including it along with the :hover pseudo-class will make sure to cover users who navigate with a mouse, as well as users that navigate with a keyboard.

Example

Color contrast ratio

It goes without saying that it is important that the text on our sites is easy to read. To achieve this, we need to make sure that the color contrast ratio (the relationship between text color and background color) is enough. But, what is "enough"? The Web Content Accessibility Guidelines (WCAG) requires text to have a minimum acceptable color contrast ratio of 3:1.

Adhering to proper color contrast ratios will help everyone, but especially users with visibility problems.

Example

Keep in mind that a ratio of 1:1 is white text on a white background (not visible), and 21:1 is black text on a white background (extremely visible).

Poor color contrast ratio (1.7:1) ↓

Lorem ipsum dolor sit amet.

Good color contrast ratio (8.3:1) ↓

Lorem ipsum dolor sit amet.
